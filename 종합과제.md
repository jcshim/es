# 📘 임베디드 시스템 프로젝트 설계 및 통합 실습 안내서

## 1. **프로젝트 개요**

임베디드 시스템 프로젝트는 센서, 마이크로컨트롤러, 액추에이터, 통신 기능을 통합하여
**하나의 완성된 시스템**을 구현하는 실습 중심 수업입니다.
학생들은 팀 또는 개인 단위로 기획 → 설계 → 제작 → 테스트 과정을 경험하며
실제 임베디드 개발 흐름을 익히는 것을 목표로 합니다.

---

## 2. **학습 목표**

* 임베디드 시스템 개발 프로세스 이해
* 하드웨어와 소프트웨어의 통합 능력 향상
* 센서/액추에이터 선택 및 인터페이스 능력 습득
* 구조적 C 프로그램 설계 및 디버깅
* 팀 커뮤니케이션 및 프로젝트 관리 능력 향상

---

## 3. **프로젝트 진행 단계**

### 3-1. **주제 선정**

예시 주제:

* 실내 환경 모니터링 시스템
* 스마트 플랜트(식물 자동 물주기)
* 자율 주행 미니카
* 얼굴 추적 회전 카메라
* 스마트 도어락

**조건**

* 2개 이상 센서 사용
* 1개 이상의 출력 장치(모터, LED, 부저 등) 포함
* 독립적으로 동작 가능한 완성도 있는 시스템
* 스마트폰의 활용
 
---

### 3-2. **요구사항 정의서 작성**

* 시스템 목적
* 입력 센서 종류
* 출력 장치 종류
* 기능 리스트(기본 3개 이상)
* 데이터 흐름(입력 → 처리 → 출력)
* 필요 부품

---

### 3-3. **시스템 설계 (H/W & S/W)**

#### (1) 하드웨어 설계

* 블록 다이어그램 작성
  (센서 → MCU → 액추에이터 → 사용자 인터페이스 구조 표시)
* 회로 구성도
* 전원 설계(USB/배터리 선택)
* 배선 안정화 계획

#### (2) 소프트웨어 설계

* 주요 기능을 상태 머신(State Machine)으로 표현
* 센서 읽기 → 로직 처리 → 출력 제어 흐름 작성
* 모듈 구조 설계
  (예: sensor.c, motor.c, display.c, main.c 등)

---

### 3-4. **모듈별 구현 및 테스트**

#### ① 센서 입력 모듈 테스트

* 센서 값 제대로 읽히는지 확인
* Serial Monitor 활용

#### ② 액추에이터 모듈 테스트

* LED, 모터, 부저 단품 테스트

#### ③ 통신 모듈 테스트 (필요 시)

* UART, I²C, SPI, Wifi
* 블루투스/와이파이 모듈

---

### 3-5. **통합(Integration) 및 디버깅**

* 모듈 간 충돌 여부 점검
* 타이머/인터럽트 사용 시 동작 타이밍 확인
* 전원 공급 부족 문제 해결
* 노이즈 및 간헐적 오동작 체크

**디버깅 방법 예시:**

* Serial.print()를 활용한 로그 출력
* 단계별 기능 활성/비활성 테스트
* 센서 필터링(평균값, 임계값 설정)

---
